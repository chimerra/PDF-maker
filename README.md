# Генератор PDF из CSV и HTML шаблонов

CLI-скрипт на Python для генерации PDF документов из CSV данных и HTML шаблонов.

## Требования

- Python 3.7+
- Библиотеки из `requirements.txt`

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Для использования `pdfkit` также потребуется установить `wkhtmltopdf`:
   - Скачайте с https://wkhtmltopdf.org/downloads.html
   - Установите и добавьте в PATH
   - Или скрипт автоматически попробует использовать `weasyprint` как альтернативу

## Структура проекта

```
.
├── pdf_generator.py    # Основной скрипт
├── run.bat            # Bat-файл для запуска
├── requirements.txt    # Зависимости Python
├── data/              # CSV файлы с данными
├── templates/         # HTML шаблоны
└── output/            # Сгенерированные PDF файлы
```

## Формат CSV файла

CSV файл должен содержать следующие колонки (с заголовками):
- `поставщик` - название поставщика (одинаковое для всех строк)
- `товар` - название товара
- `цена` - цена товара
- `количество` - количество товара

Пример:
```csv
поставщик,товар,цена,количество
ООО "Поставщик А",Товар 1,1500.50,10
ООО "Поставщик А",Товар 2,2300.00,5
```

## HTML шаблон

HTML шаблон должен содержать следующие плейсхолдеры:

- `{{contractor}}` - заменяется на название поставщика
- `{{product}}` - заменяется на название товара (в строке таблицы)
- `{{price}}` - заменяется на цену (в строке таблицы)
- `{{qty}}` - заменяется на количество (в строке таблицы)
- `{{index}}` - (опционально) заменяется на номер строки
- `{{total}}` - (опционально) заменяется на сумму (цена × количество)

Важно: В шаблоне должна быть строка таблицы (`<tr>...</tr>`) с плейсхолдерами `{{product}}`, `{{price}}`, `{{qty}}`. Эта строка будет использована как шаблон для генерации всех строк из CSV.

## Использование

### Через bat-файл (Windows):
```bash
run.bat
```

### Напрямую через Python:
```bash
python pdf_generator.py
```

## Процесс работы

1. Скрипт сканирует директории `data/` и `templates/`
2. Выводит пронумерованные списки доступных файлов
3. Пользователь выбирает CSV файл и HTML шаблон по номеру
4. Скрипт читает данные из CSV
5. Генерирует HTML, подставляя данные в шаблон
6. Конвертирует HTML в PDF
7. Сохраняет PDF в директорию `output/` с именем `nakladnaya_YYYYMMDD_HHMMSS.pdf`
8. Автоматически открывает сгенерированный PDF

## Обработка ошибок

Скрипт обрабатывает следующие ошибки:
- Отсутствие CSV файлов или HTML шаблонов
- Неправильный формат CSV (отсутствие необходимых колонок)
- Ошибки чтения файлов
- Отсутствие библиотек для генерации PDF (pdfkit/weasyprint)
- Ошибки генерации PDF

## Примечания

- Если `pdfkit` недоступен, скрипт автоматически попробует использовать `weasyprint`
- Для работы `pdfkit` требуется установленный `wkhtmltopdf`
- `weasyprint` не требует дополнительных зависимостей, но может иметь ограничения в стилизации
